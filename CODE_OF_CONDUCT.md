# Code of Conduct

This project values clear thinking, good engineering, and shipping working software. These are the standards we hold ourselves to.

## Write Good Code

- Test what you ship. If it's not tested, it's not done.
- Read the existing code before changing it. Understand the design before proposing alternatives.
- Keep changes focused. A bug fix is a bug fix, not a refactoring opportunity.
- Prefer simple solutions. Don't build abstractions for problems you don't have yet.

## Respect Scope

- Solve the problem at hand. Don't gold-plate.
- If a PR grows beyond its original purpose, split it.
- Design decisions are documented for a reason. Challenge them with evidence, not preference.

## Communicate Clearly

- Say what you mean. Be direct.
- Disagree with ideas, not people. "This approach has a race condition" is useful. Personal remarks are not.
- If you're stuck, say so. If something is broken, say so. Clarity over ego.

## That's It

No lengthy process. Write good code, keep scope tight, communicate like an adult. The work speaks for itself.
